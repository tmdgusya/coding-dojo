## RED → GREEN 학습 방식 (TDD 기반)

모든 과제는 **테스트 주도 개발(TDD)** 방식으로 구성됩니다:

1. **RED (실패하는 테스트)**
   - 테스트 코드를 먼저 작성합니다.
   - 구현 코드에는 `todo!()` (Rust) 또는 `raise NotImplementedError` (Python) 등의 플레이스홀더를 둡니다.
   - 처음 `cargo test` 또는 `pytest`를 실행하면 **모든 테스트가 실패**합니다.

2. **GREEN (테스트 통과)**
   - 제자가 직접 구현 코드를 작성하여 테스트를 통과시킵니다.
   - 임무별로 테스트를 실행할 수 있도록 구성합니다. (예: `cargo test mission_1`)

3. **REFACTOR (리팩토링)**
   - 테스트가 통과한 후, 코드를 더 깔끔하게 개선합니다.

### 과제 구조 예시

```
{project}-dojo/
├── src/
│   └── lib.rs (또는 main.py)    # todo!() 플레이스홀더가 있는 구현 파일
├── tests/
│   └── mission_tests.rs          # 미리 작성된 테스트 (처음엔 모두 실패)
├── docs/
│   ├── README.md                 # 임무 설명, 예상 출력, 학습 경로
│   └── Assistance.md             # 조수 지시서 (힌트 제공 방법)
└── README.md                     # 빠른 시작 가이드
```

### 테스트 작성 원칙

- 각 임무(Mission)별로 테스트를 그룹화합니다.
- 테스트 이름은 `mission_N_기능명` 형식으로 작성합니다.
- 테스트는 **예상 출력 결과를 명확하게** 보여줘야 합니다.
- 제자가 `cargo test mission_1` 처럼 특정 임무만 테스트할 수 있어야 합니다.
