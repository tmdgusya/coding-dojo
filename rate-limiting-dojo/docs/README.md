# Rate Limiting Dojo

## 목표
이 수련장은 분산 환경에서의 처리량 제한(Rate Limiting) 알고리즘을 직접 구현하고 마스터하기 위해 만들어졌습니다.
Roach 님은 이 과정을 통해 단순히 라이브러리를 사용하는 것을 넘어, 원리를 깊이 이해하고 구현할 수 있는 능력을 갖추게 될 것입니다.

## 임무 1: 고정 윈도우 카운터 (Fixed Window Counter) - 완료

## 임무 2: 슬라이딩 윈도우 로그 (Sliding Window Log)

### 설명
고정 윈도우 방식의 단점인 경계 지점의 트래픽 버스트(Burst) 문제를 해결합니다. 
모든 요청의 타임스탬프를 기록하고, 현재 시점을 기준으로 과거 윈도우 시간 내에 몇 개의 로그가 있는지 확인하여 제한합니다.

### 요구사항
1. `SlidingWindowLogRateLimiter` 클래스를 구현하세요.
2. `storage`에는 유저별로 타임스탬프 리스트를 저장합니다.
3. 요청이 들어올 때마다:
    - 윈도우 범위 밖의 낡은 타임스탬프를 제거합니다.
    - 현재 타임스탬프를 추가합니다.
    - 리스트의 크기가 `max_requests`를 넘는지 확인합니다.
4. 테스트 코드를 통해 경계 조건에서도 정확히 제한되는지 확인하세요.

### 예상 출력 결과
```text
[시간 0.1s] User_A 요청 1: 허용됨
[시간 0.5s] User_A 요청 2: 허용됨
[시간 0.9s] User_A 요청 3: 허용됨
[시간 1.05s] User_A 요청 4: 허용됨 (윈도우가 0.05s ~ 1.05s로 슬라이딩)
```

## 사용법
`uv`를 사용하여 가상 환경을 관리하고 테스트를 실행합니다.
```bash
uv run pytest test/
```
